# 2-3歳モビコール導入・維持期 日誌と医師サマリー設計

作成日: 2026-04-28

対象: 2-3歳の小児慢性機能性便秘症。モビコールなどのPEG製剤を導入中、または維持療法中の児。

目的:

- 保護者が毎日無理なく入力できる便秘日誌を設計する。
- 医師が短時間で治療経過、反応性、再燃兆候、安全面を確認できるサマリーを生成する。
- AIは診断、処方量変更、治療中止判断を行わず、日誌整理、受診目安提示、医師相談の準備を支援する。

根拠として反映する2025年版ガイドライン要点:

- 便塞栓の有無を判定し、便塞栓がある場合は便塞栓除去を行う。
- 便塞栓除去後、または便塞栓のない児には維持療法を開始する。
- 維持療法の薬物療法としてPEG製剤を推奨する。
- 維持療法では排便習慣の指導を行う。
- トイレトレーニングは、排便の苦痛が軽減してから開始または再開する。
- 治療薬の減量、中止、治療中止による再燃の有無を評価するため、定期受診を推奨する。
- 標準的な維持療法で改善しない場合は専門医療機関への紹介を推奨する。
- 便塞栓除去の効果判定では、臨床症状、身体所見、排便回数、便形状、画像所見の変化を確認する。

## 1. 設計方針

このAIは「便秘診療を置き換えるAI」ではなく、「家庭内観察を医療者が使える形に整えるAI」として設計する。

保護者向けには、安心できる言葉で、今日入力すべき項目、継続観察の意味、受診相談の目安を示す。医師向けには、感情表現を省き、期間、排便回数、便形状、痛み、がまん、便失禁、服薬状況、再燃兆候を構造化して提示する。

## 2. フェーズ定義

### A. 導入期

定義:

- モビコール開始直後から、便性状と排便リズムが安定するまで。
- 便塞栓除去後の維持療法開始直後もここに含める。

主な観察目的:

- 排便が出始めたか。
- 便が硬すぎないか、またはゆるすぎないか。
- 腹痛、嘔吐、強い腹部膨満などがないか。
- 排便時痛や排便がまんが軽減しているか。
- 服薬が継続できているか。

AIの役割:

- 入力漏れを補う。
- 「薬の量を変える」ではなく「処方医に相談する材料」を整理する。
- 強い症状、心配な変化、服薬困難がある場合は受診相談を促す。

### B. 維持期

定義:

- 便塞栓がなく、モビコールなどの維持療法で排便リズムを保っている期間。

主な観察目的:

- 排便回数と便形状が安定しているか。
- 排便時痛、便意がまん、便失禁が減っているか。
- 減量、中止、飲み忘れ後に再燃していないか。
- トイレトレーニングを開始または再開できる状態か。

AIの役割:

- 週単位で安定性を評価する。
- 再燃兆候を早めに拾う。
- トイレトレーニングは「痛みが軽減してから」という原則を守る。
- 定期受診前に医師用サマリーを作る。

## 3. 毎日の日誌項目

### 必須項目

| 項目 | 入力形式 | 医師サマリーでの使い道 |
|---|---|---|
| 排便の有無 | あり/なし | 排便間隔、無排便日数 |
| 排便回数 | 数値 | 週あたり回数、導入後の反応 |
| 便形状 | Bristol便形状スケール相当、または「硬い/普通/泥状/水様」 | 便が硬い、ゆるい、安定の評価 |
| 排便時痛 | なし/少し/強い/泣く | 排便がまんの悪循環評価 |
| 排便がまん | なし/あり/強い | 痛みと行動の関連評価 |
| 便失禁・下着汚れ | なし/少量/多い | 便塞栓や再燃の手がかり |
| 腹痛・腹部膨満 | なし/軽い/強い | 安全面、受診相談判断 |
| 嘔吐 | なし/あり | 受診相談判断 |
| 食欲・機嫌 | いつも通り/低下 | 全身状態の変化 |
| モビコール内服 | 処方通り/一部飲めた/飲めなかった | 効果不十分時の解釈 |

### 任意項目

| 項目 | 入力形式 | 使い道 |
|---|---|---|
| トイレ・おまるに座れたか | なし/座れた/嫌がった | 排便習慣指導、トイトレ再開判断 |
| 排便場所 | おむつ/トイレ/おまる/その他 | 発達段階と習慣化 |
| 出血 | なし/紙に少量/便に混じる | 裂肛、硬便、医師相談材料 |
| 浣腸・坐薬・他の薬 | なし/あり、内容 | 追加介入の把握 |
| 水分・食事の大きな変化 | 自由記載 | 一時的変化の解釈 |
| 発熱・感染症 | なし/あり | 食欲低下や便性状変化の背景 |
| 保護者メモ | 自由記載 | 心配事、園での様子 |

## 4. 週次集計項目

AIは7日単位で以下を自動集計する。

- 排便ありの日数。
- 総排便回数。
- 最長無排便日数。
- 硬便の日数。
- 水様便または泥状便の日数。
- 排便時痛があった日数。
- 排便がまんがあった日数。
- 便失禁または下着汚れがあった日数。
- モビコールを処方通り飲めた日数。
- 腹痛、腹部膨満、嘔吐、食欲低下があった日数。
- トイレまたはおまるに座れた日数。

週次コメントは、保護者向けと医師向けで分ける。

保護者向け:

- 「この1週間は排便が5日あり、痛みの記録は1日でした。前週より痛みが減っています。」
- 「無排便が続いています。処方内容の変更はAIでは判断できないため、日誌を持って医師に相談してください。」

医師向け:

- 「7日間で排便5日、総排便6回。硬便2日、水様便0日。排便時痛1日、がまん1日。PEG内服は7/7日。」

## 5. 医師用サマリー設計

### 基本構成

1. 患児情報
2. 治療フェーズ
3. 集計期間
4. 現在の処方・内服状況
5. 排便状況
6. 症状・行動
7. 安全面
8. トイレトレーニング状況
9. 保護者の主な心配
10. AIによる要確認ポイント

### 医師用サマリーテンプレート

```text
【小児便秘 日誌サマリー】

対象: 2-3歳 / モビコール導入期または維持期
集計期間: YYYY-MM-DD から YYYY-MM-DD
日誌入力日数: X/Y日

1. 治療フェーズ
- フェーズ: 導入期 / 維持期 / 減量・中止後観察 / 不明
- 便塞栓除去後: はい / いいえ / 不明
- 前回受診日: YYYY-MM-DD / 不明
- 次回受診予定: YYYY-MM-DD / 未定

2. 内服状況
- モビコール: 処方通り X/Y日、一部内服 X/Y日、内服不可 X/Y日
- 飲みにくさ: なし / あり
- 追加薬・処置: なし / あり（内容）

3. 排便状況
- 排便ありの日数: X/Y日
- 総排便回数: X回
- 最長無排便: X日
- 便形状: 硬便 X日、普通便 X日、泥状・水様 X日
- 便失禁・下着汚れ: X日

4. 症状・行動
- 排便時痛: X日
- 排便時に泣く/強く嫌がる: X日
- 排便がまん: X日
- 出血: なし / あり（頻度・程度）
- 腹痛・腹部膨満: なし / あり（頻度・程度）
- 嘔吐: なし / あり
- 食欲・機嫌低下: なし / あり

5. 排便習慣・トイレトレーニング
- トイレ/おまる着座: X/Y日
- 着座への抵抗: なし / あり
- 排便時痛が残るため、トイレトレーニング開始/再開は慎重に判断が必要: はい / いいえ

6. 保護者の主な心配
- 自由記載から要約

7. AIから医師への要確認ポイント
- 無排便、硬便、痛み、がまん、便失禁、内服困難、再燃兆候などから該当項目を列挙
- AIは処方変更判断を行っていない
```

## 6. AI判定ラベル

AIは診断名や処方変更を出さず、以下のような観察ラベルを付ける。

| ラベル | 条件の例 | AIの返答方針 |
|---|---|---|
| 安定傾向 | 排便が定期的、痛み少ない、内服継続 | 継続記録と定期受診準備 |
| 硬便・痛み残存 | 硬便、痛み、がまんが続く | 医師相談用に頻度を整理 |
| ゆるすぎる便 | 泥状・水様便が続く | 脱水や体調変化を確認し、医師相談 |
| 内服困難 | 飲めない日が複数 | 飲めない理由を整理し処方医へ相談 |
| 再燃疑い | 減量・中止・飲み忘れ後に硬便、無排便、痛み、がまんが増える | 早めの医師相談を促す |
| 便塞栓疑いの相談材料あり | 無排便、便失禁、腹部膨満、強いがまんなどが組み合わさる | 受診相談を促し、医師用サマリーに明示 |
| 要早期相談 | 嘔吐、強い腹痛、著しい腹部膨満、全身状態低下など | 速やかな医療相談を促す |

注意:

- レッドフラッグの最終定義は医療者監修項目として別途確定する。
- AIは「便塞栓です」「薬を増やしてください」「中止してよいです」とは言わない。

## 7. 保護者向け応答方針

### 言ってよいこと

- 日誌の入力を促す。
- 排便回数、便の硬さ、痛み、がまん、便失禁の変化を説明する。
- 「処方医に相談する材料がそろっています」と伝える。
- 定期受診で見せるサマリーを作る。
- 排便時痛が残る時期は、トイレトレーニングを急がないよう伝える。

### 言わないこと

- モビコールの用量を増減する具体指示。
- 治療中止の指示。
- 診断の確定。
- 便塞栓除去の家庭実施指示。
- 他薬剤への変更指示。

### 応答例

```text
この3日間は排便がなく、昨日からお腹の張りと排便をがまんする様子が記録されています。
モビコールの量をAIが判断して変えることはできませんが、医師に相談する価値のある変化です。
直近7日分の排便回数、便の硬さ、痛み、内服状況をまとめて受診時に見せられるようにします。
```

```text
今週は排便が6日あり、痛みの記録はありませんでした。
痛みが減っている時期は、食後など決まった時間に短く座る練習を検討しやすい状態です。
嫌がる場合は無理に続けず、次回受診でタイミングを相談してください。
```

## 8. 状態遷移モデル

```text
未評価
  ↓
初期評価
  ↓
便塞栓ありの可能性 → 医師評価・便塞栓除去 → 導入期
  ↓
便塞栓なし/除去後
  ↓
モビコール導入期
  ↓
維持期
  ↓
安定継続 ── 減量・中止後観察
  ↓              ↓
再燃兆候 ←──────
  ↓
医師相談・治療方針再評価
```

AIが状態遷移を確定するのではなく、日誌から「現在どのフェーズとして扱うと整理しやすいか」を仮ラベル化する。

## 10. 最小実装の画面構成

### 保護者向け日誌画面

- 今日の排便: あり/なし
- 回数
- 便の硬さ
- 痛み
- がまん
- 下着汚れ
- お腹の張り/腹痛
- 嘔吐
- 食欲・機嫌
- モビコールを飲めたか
- メモ

2-3歳では、入力負担を下げるため、初期画面は10項目以内にする。出血、トイレ着座、追加薬は「詳しく記録する」に入れる。

### 医師サマリー画面

- 7日/14日/30日を切り替え。
- 排便回数、硬便、痛み、がまん、便失禁、内服状況を表で表示。
- AI要約は短文で表示。
- PDFまたはテキストで出力できる形式にする。

## 11. 次に決めるべき医療者監修項目

- レッドフラッグの具体リスト。
- 早期受診、通常受診、経過観察の分類基準。
- 医師用サマリーに出す期間の標準値、7日/14日/30日のどれを既定にするか。
- モビコールの処方量表示を、袋数、包数、g、回数のどの形式で入力させるか。
- 便形状の表現をBristol風にするか、保護者向けの日本語4分類にするか。
- 園での排便、トイレ拒否、便失禁をどの粒度で記録するか。
- 神経発達症の特性がある児への入力支援とサマリー補足。
