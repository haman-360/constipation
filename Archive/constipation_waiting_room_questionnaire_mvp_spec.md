# 2-3歳便秘AI 待合アンケート 最小プロトタイプ仕様

作成日: 2026-04-28

## 目的

2-3歳の便秘診療で、待合中に保護者が短時間で入力でき、医師が診察前に状態を把握できる最小プロトタイプを作る。

この仕様は、既存の `constipation_waiting_room_questionnaire_v2.md`、`constipation_waiting_room_questionnaire_v2_implementation_spec.md`、`constipation_waiting_room_questionnaire_v2_summary_logic.md` から、初回実装に必要な範囲だけを切り出したもの。

## AIの役割

AIがすること:

- 保護者の回答を構造化する。
- 便秘の状態、内服状況、安全面の確認事項を診察前に整理する。
- 医師が追加確認するとよい観察情報を列挙する。
- 強い症状、嘔吐、食欲・機嫌の変化がある場合、受付または医療スタッフにも伝えるよう案内する。

AIがしないこと:

- 診断。
- 便塞栓の有無判断。
- モビコールなどの処方量変更。
- 治療中止可否の判断。
- 専門紹介の要否判断。
- 先天性疾患の有無判断。

## MVPの範囲

### 必ず聞く基本6問

1. 最後にうんちが出たのはいつか。
2. 最近、うんちはどれくらいの間隔で出ているか。
3. 最近のうんちの硬さ。
4. うんちをするとき痛がるか。
5. うんちをがまんしている様子があるか。
6. 便秘のお薬を今どのように飲んでいるか。

### 条件付きで聞く追加質問

MVPでは、全20問を実装しない。基本6問の回答から、診察前に必要性が高い確認だけを自動展開する。

| 追加確認 | 表示条件 |
|---|---|
| お腹の張りや腹痛 | 4日以上うんちが出ていない、または水のような便 |
| 吐いたこと | 4日以上うんちが出ていない、強い腹痛・お腹の張り、水のような便 |
| 食欲・機嫌 | 4日以上うんちが出ていない、嘔吐あり、水のような便、強い腹痛・お腹の張り |
| 血がつくこと | 硬い便、強い痛み、泣く・とても嫌がる |
| パンツやおむつに少し便がつくこと | がまんあり、強い痛み、医師が確認したい設定の場合 |
| お薬が飲みにくい理由 | お薬が飲みにくくて残る、または追加チェックで飲みにくさあり |
| お薬が少なめ・飲み忘れ・中止後の変化 | 最近少なめ、飲み忘れ、中止中、または追加チェックで飲み忘れあり |

## 画面構成

### 画面1: 開始

タイトル:

```text
うんちの様子を教えてください
```

説明:

```text
診察の前に、最近のうんちの様子を確認します。
わかる範囲で、近いものを選んでください。
薬の量をこの画面で決めることはありません。
```

ボタン:

```text
はじめる
```

### 画面2: 基本入力

基本6問を1問ずつ、または1ページまとめ入力で表示する。待合スマホでは1問1画面、タブレットではまとめ入力でもよい。

### 画面3: 追加確認

基本6問の回答に応じて必要な追加質問だけを表示する。

追加質問がない場合はスキップする。

### 画面4: 終了

表示:

```text
ありがとうございました。
入力内容を診察前の確認に使います。
```

安全面の注意表示:

```text
強い腹痛、吐いた、食欲や機嫌がいつもと違う場合は、入力だけでなく受付または医療スタッフにもお伝えください。
```

## フィールド定義

### 基本6問

| ID | 質問 | 型 | 選択肢 |
|---|---|---|---|
| q1_last_bowel_movement | 最後にうんちが出たのはいつですか？ | single_select | 今日 / 昨日 / 2-3日前 / 4日以上前 / わからない |
| q2_bowel_frequency | 最近、うんちはどれくらいの間隔で出ていますか？ | single_select | 1日に2回以上 / 1日1回くらい / 2-3日に1回くらい / 4日以上あくことがある / わからない |
| q3_stool_consistency | 最近のうんちの硬さはどれに近いですか？ | single_select | 硬い / 普通 / やわらかい / 水のよう / わからない |
| q4_pain | うんちをするとき、痛がることはありますか？ | single_select | 痛がらない / 少し痛がる / 強く痛がる / 泣く、またはとても嫌がる / わからない |
| q5_withholding | うんちをがまんしているように見えることはありますか？ | single_select | ない / ある / 強くある / わからない |
| q6_med_status | 便秘のお薬は、今どのように飲んでいますか？ | single_select | 先生に言われた量で飲んでいる / 先生から調節してよいと言われていて、うんちの様子を見ながら調節している / 調節してよいと言われたが、最近は少なめになっている / ときどき忘れる / 飲みにくくて残る / 今は中止している / 便秘のお薬は使っていない / わからない |

| ID | 質問 | 型 | 選択肢 |
|---|---|---|---|

| q10_vomiting | 吐いたことはありますか？ | single_select | ない / ある / わからない |
| q11_appetite_mood | 食欲や機嫌はいつも通りですか？ | single_select | いつも通り / 食欲が少ない / 機嫌が悪い / 食欲も機嫌も気になる / わからない |

## 表示条件ロジック

### 4日以上無排便

条件:

- `q1_last_bowel_movement = 4日以上前`
- または `q2_bowel_frequency = 4日以上あくことがある`

表示:

- `q9_abdominal_symptom`
- `q10_vomiting`
- `q11_appetite_mood`

### 硬便・痛み・がまん

条件:

- `q3_stool_consistency = 硬い`
- または `q4_pain = 強く痛がる`
- または `q4_pain = 泣く、またはとても嫌がる`
- または `q5_withholding = ある`
- または `q5_withholding = 強くある`

表示:

- `q7_blood`
- `q8_soiling`

### 水のような便

条件:

- `q3_stool_consistency = 水のよう`

表示:

- `q9_abdominal_symptom`
- `q10_vomiting`
- `q11_appetite_mood`

### 内服困難

条件:

- `q6_med_status = 飲みにくくて残る`
- または `q6_med_adherence_flags` に `飲みにくくて残る` が含まれる

表示:

- `q13_med_difficulty_reason`

### お薬が少なめ・飲み忘れ・中止

条件:

- `q6_med_status = 調節してよいと言われたが、最近は少なめになっている`
- または `q6_med_status = ときどき忘れる`
- または `q6_med_status = 今は中止している`
- または `q6_med_adherence_flags` に `ときどき忘れる` が含まれる

表示:

- `q14_change_after_less_med`

### 医師指示範囲内の自己調節

条件:

- `q6_med_status = 先生から調節してよいと言われていて、うんちの様子を見ながら調節している`

扱い:

- これだけでは `q14_change_after_less_med` を必須表示しない。
- ただし追加チェックで飲み忘れがある、または別UIで「最近少なめ」を選べる場合は `q14_change_after_less_med` を表示する。

## 医師向けミニサマリー

MVPでは、最初から完全版サマリーを出さず、診察前に重要な項目を短く出す。

```text
【診察前 便秘ミニサマリー】

最終排便:
最近の排便頻度:
うんちの硬さ:
排便時痛:
がまん:

お薬:
- 状況:
- 補足:
- 少なめ・飲み忘れ・中止後の変化:
- 飲みにくさ:

追加確認:
- お腹の張り・腹痛:
- 嘔吐:
- 食欲・機嫌:
- 血がつく:
- パンツやおむつに少し便がつく:

AI追加確認候補:
-

AIが判断していないこと:
- 診断
- 便塞栓の有無
- モビコールなどの処方量変更
- 治療中止可否
- 専門紹介の要否
```

### 未回答項目の扱い

- 分岐で表示されなかった項目は `未確認` とする。
- 保護者が `わからない` を選んだ場合は `わからない` とする。
- 該当しない項目は `該当なし` とする。

## AI追加確認候補の生成ルール

### 4日以上無排便または間隔延長

条件:

- `q1_last_bowel_movement = 4日以上前`
- または `q2_bowel_frequency = 4日以上あくことがある`

出力例:

```text
- 4日以上うんちが出ていない、または4日以上あくことがあるため、お腹の張り・腹痛、嘔吐、食欲・機嫌を診察で確認。
```

### 硬便・痛み・がまん

条件:

- 硬い便、強い痛み、泣く・とても嫌がる、がまんあり、のいずれか。

出力例:

```text
- 硬い便、痛み、がまんの組み合わせを確認。出血や便が少しつく様子もあわせて確認。
```

### 水のような便

条件:

- `q3_stool_consistency = 水のよう`

出力例:

```text
- 水のような便あり。お腹の張り・腹痛、嘔吐、食欲・機嫌の変化をあわせて確認。
```

### お薬が少なめ・飲み忘れ・中止

条件:

- 最近少なめ、飲み忘れ、中止中、のいずれか。

出力例:

```text
- 実際のお薬が少なめ、飲み忘れ、または中止中。以後の硬さ、間隔、痛みの変化を確認。
```

### お薬が飲みにくい

条件:

- 飲みにくくて残る、または追加チェックで飲みにくさあり。

出力例:

```text
- お薬が飲みにくい可能性あり。味、量、タイミング、子どもの拒否など具体的な理由を確認。
```

## 保護者向け安全表示

次のいずれかがある場合、終了画面で強調表示する。

- `q9_abdominal_symptom = 強い`
- `q10_vomiting = ある`
- `q11_appetite_mood = 食欲が少ない`
- `q11_appetite_mood = 機嫌が悪い`
- `q11_appetite_mood = 食欲も機嫌も気になる`

表示文:

```text
強い腹痛、吐いた、食欲や機嫌がいつもと違う場合は、入力だけでなく受付または医療スタッフにもお伝えください。
```

## プロトタイプの非対象

初回プロトタイプでは、以下は実装しないか、管理画面側の任意項目として扱う。

- 周産期・新生児期の背景情報。
- 基礎疾患、発達・行動面、定期薬。
- 家族歴。
- トイレ詰まり。
- 園・外出先での排便困難。
- 浣腸歴、受診中断歴、便秘開始時期。

これらは初回実装後、医師レビューで必要性が高い項目から追加する。

## 実装順

1. 基本6問の入力フォームを作る。
2. 表示条件ロジックで追加質問を出す。
3. 入力データをJSONとして保存できるようにする。
4. 医師向けミニサマリーを生成する。
5. 安全表示の条件を実装する。
6. 既存テストケースのうち、TC-AからTC-EをMVP項目に合わせて確認する。
