# 年齢層別ChatGPT相談プロンプト設計

## 目的

0-1歳、2-3歳、4歳以降で、ChatGPTに整理させる観点を切り替える。

共通部分は維持し、年齢層ごとの追加観点だけを差し替えることで、現MVPの2-3歳ロジックを壊さずに拡張する。

この文書は実装前の設計たたき台であり、診断、処方、紹介判断をChatGPTに代替させるものではない。

## 共通ルール

全プロファイル、全モードで共通する。

- 診断や処方の最終決定は医師が行う。
- 患者・保護者へ直接、薬の増減・中止・再開を指示しない。
- 具体的な処方量変更の数値指示は出さない。
- 不足情報と不確実性を明示する。
- 外部Web検索やURL引用は不要とする。
- 提示された経過、医師入力、日誌、一般的診療原則に基づいて整理する。
- 患者・保護者を責める表現を避ける。
- 方針候補を出す場合も、根拠、前提、不確実性、追加確認事項を併記する。

## モード別の共通設計

### 診察前整理モード

目的:

- 診察前に、変化点、不足情報、矛盾点、追加確認事項を整理する。
- 治療方針の推奨は行わない。

出力:

1. 現在の状態の要約
2. 前回から改善した点
3. 前回から悪化した点
4. 年齢層別に医師が確認すべき追加情報
5. 矛盾または不足している情報
6. 診察で優先して確認する候補

### 医師向け治療方針検討モード

目的:

- 医師が診察で検討するための方針候補を整理する。
- 患者・保護者へ直接指示する文体にはしない。

出力:

1. 現在の状態評価
2. 方針候補と推奨度（推奨 / 検討可 / 慎重 / 避けたい）
3. 各方針の理由
4. 避けたい方針とその理由
5. 次回再診間隔の考え方
6. 追加で確認すべき情報
7. 保護者が不安な場合の中立的な説明文案

## 年齢プロファイル別の追加観点

### 0-1歳: infant

診察前整理モードに追加する観点:

- 出生時や1か月健診で指摘があったか。
- 出生直後または生後早期から便秘が続いているか。
- 哺乳量、食事量、体重増加に気になる点があるか。
- 嘔吐、腹部膨満、発熱、活気低下があるか。
- 離乳食開始やミルク変更と便秘の時期関係があるか。
- 牛乳・乳製品、母の乳製品摂取、ミルク変更と、便秘、湿疹、血便、吐き戻しの関連が疑われる訴えがあるか。
- 浣腸、綿棒刺激、市販薬の使用頻度が不明でないか。
- 肛門周囲や肛門の開口部位など、診察で確認すべき身体所見が残っていないか。

治療方針検討モードに追加する観点:

- 薬の方針候補を出す前に、哺乳、体重、嘔吐、腹部膨満、発症時期、出生時・1か月健診の指摘が十分に確認されているかを整理する。
- 牛乳アレルギーを断定せず、牛乳・乳製品やミルク変更との関連があれば追加確認候補として扱う。
- 浣腸、綿棒刺激、市販薬について、具体的な頻度指示は出さない。
- 保護者説明案は、自己判断ではなく、記録と診察相談につなげる文体にする。

避けたい出力:

- 胎便遅延や先天性疾患を断定する。
- 牛乳アレルギーと断定する。
- 浣腸や綿棒刺激の具体的な回数を指示する。
- 薬の量を具体的に増減する。

### 2-3歳: toddler

診察前整理モードに追加する観点:

- 排便頻度、便の硬さ、排便時痛、がまんの変化。
- 薬の飲み忘れ、少なめ、中止、飲みにくさと便性状の関係。
- トイレトレーニング状況と痛み、がまん、便失禁の関係。
- 直近日誌の排便日数、硬便日数、痛みの日、内服できた日。
- 保護者が薬の継続、減量、中止にどの程度不安を持っているか。

治療方針検討モードに追加する観点:

- 長期安定例では、段階的減量を独立した方針候補として必ず評価する。
- 同量継続は、保護者不安への短期的な橋渡しとして扱い、漫然継続や長すぎる再診間隔とは分ける。
- 減量や飲み忘れ後に硬便、痛み、がまんが再燃している場合は、急な中止や減量継続を慎重または避けたい方針として整理する。
- トイレトレーニングの進み具合と痛み、がまんを分けて整理する。

避けたい出力:

- `1包に減量` のような具体的処方指示。
- `中止してよい` のような断定。
- 保護者不安を軽視する表現。
- トイレトレーニングを急がせる表現。

### 4歳以降: child

診察前整理モードに追加する観点:

- 園・学校で排便できるか。
- 便失禁、下着汚れ、本人の自覚があるか。
- 夜尿、昼間尿失禁があるか。
- 腹痛、食欲低下、嘔吐、腹部膨満があるか。
- 偏食、水分不足、朝のトイレ時間不足、生活リズムの乱れがあるか。
- 発達相談、他院通院中の病気、他院処方、長期管理薬があるか。
- 便秘が怖くて薬をやめられない希望があるか。
- 頓用薬を家に置きたい希望は、患者画面ではなく医師側確認事項として扱う。
- 本人と保護者の困りごとや目標がずれていないか。

治療方針検討モードに追加する観点:

- 便失禁や学校での排便回避がある場合、薬の方針だけでなく生活場面の支援も検討材料として整理する。
- 発達相談、他院通院中の病気、他院処方薬は、便秘との関係を断定せず、不足情報として明示する。
- 便秘が怖くて薬をやめられない場合は、実際の便性状、排便頻度、便失禁、腹痛、薬の使用状況を分けて整理する。
- 漫然継続、不安への短期的対応、頓用薬の位置づけ、保険診療上の妥当性を分けて整理する。
- 医療側、患児、保護者の治療方針に一致性が見られない場合は、当院での治療継続困難または二次医療機関紹介を検討する論点として整理する。
- 学校や園での排便困難は、本人を責めず、環境調整や相談材料として扱う。

避けたい出力:

- 便失禁、夜尿、昼間尿失禁を本人の努力不足として扱う。
- 発達特性や他院通院中の病気が便秘の原因だと断定する。
- 薬を家に置きたい希望を一律に否定する。
- 具体的な薬の増減量、減量幅、中止時期を指示する。

## 実装時の差し込み方

将来的に `age_profile` を実装したら、ChatGPT貼り付け用テキストの冒頭に以下を追加する。

```text
【年齢プロファイル】
対象年齢層: infant / toddler / child
年齢表示: 〇歳〇か月

【この年齢層で特に確認する観点】
...
```

実装時の想定:

- `patients.birth_date` と直近受診日から年齢を計算する。
- 年齢から `age_profile` を決める。
- `generatePreVisitContext_` と `generateTreatmentReviewContext_` に、年齢層別の追加観点を差し込む。
- 年齢不明時は `toddler` 互換の安全側表示にし、`年齢未確認` を明示する。

## 実装前レビュー項目

- 0-1歳で、牛乳アレルギーや身体所見の扱いが診断誘導になっていないか。
- 2-3歳で、現MVPの安全な表現を壊していないか。
- 4歳以降で、便失禁、夜尿、発達相談、頓用薬希望の聞き方が本人や保護者を責める表現になっていないか。
- 治療方針検討モードで、推奨度が出ても具体的な処方指示になっていないか。
- 保護者説明案が、自己判断ではなく診察相談につながる文体になっているか。
