# 年齢層拡張設計

## 目的

現MVPの2-3歳向け便秘問診を土台に、0-1歳、4歳以降へ拡張できるようにする。

0から別アプリを作り直すのではなく、患者ID、QR作成、問診送信、Google Sheets保存、便秘履歴、医師入力、ChatGPT貼り付け用テキスト生成は共通化し、年齢層ごとの問診項目と医師向け整理ロジックだけを切り替える。

この文書は実装前の設計たたき台であり、年齢層別の医療内容は医療者レビューで確定する。

## 基本方針

- 患者画面では年齢を自由入力させない。
- 年齢は `patients.birth_date` と受診日から医師側で自動計算する。
- 初診では、生年月日が未登録でも問診入力は進められる。
- 初診後に医師またはスタッフが `patients.birth_date` を1回入力する。
- 再診では、便秘履歴とChatGPT貼り付け用テキストに受診日時点の年齢を表示する。
- QR読み込み直後に年齢を表示する機能は、問診開始前にWeb Appへ照会が必要になるため現MVPでは見送る。

## 年齢プロファイル

| プロファイル | 想定年齢 | 位置づけ                                                            | 実装優先度 |
| ------------ | -------: | ------------------------------------------------------------------- | ---------- |
| infant       |    0-1歳 | 乳児便秘。背景疾患、哺乳、離乳食、体重増加などの確認が重要。        | 次期設計   |
| toddler      |    2-3歳 | 現MVP。便性状、痛み、がまん、薬の飲み方、トイレトレーニングが中心。 | 実装済み   |
| child        |  4歳以降 | 園・学校生活、排便回避、便失禁、生活習慣、服薬自己管理が重要。      | 次期設計   |

実装上は、計算された年齢から内部的に `age_profile` を決める。

```text
0歳0か月-1歳11か月: infant
2歳0か月-3歳11か月: toddler
4歳0か月以上: child
年齢不明: toddler互換の安全側表示 + 年齢未確認
```

## 共通データ

全プロファイルで共通して保存する。

- `patient_id`
- `visit_id`
- `visit_token`
- `submitted_at`
- 最終排便
- 排便頻度
- 便の硬さ
- 排便時痛
- がまん
- 便秘薬の内服状況
- 日誌サマリー
- 患者向けメモ
- 医師向けサマリー

## 年齢層別に切り替える項目

### 0-1歳: infant

医療者レビュー用の確認候補:

- 胎便排泄時期
- 出生後早期からの便秘か
- 哺乳量、哺乳不良
- 体重増加不良
- 嘔吐、胆汁性嘔吐
- 腹部膨満
- 血便
- 発熱、活気不良
- 離乳食開始時期と食事変化
- ミルク変更、牛乳・乳製品、母の乳製品摂取と症状の関係
- 肛門周囲の気になる所見、肛門の開口部位
- 便秘薬、浣腸、綿棒刺激などの使用状況

画面設計の方向:

- トイレトレーニング項目は出さない。
- 保護者が答えやすい生活情報を中心にする。
- 先天性疾患や全身状態に関わる情報は、診断ではなく「医師が確認すべき情報」として整理する。
- 肛門の開口部位や計測の要否は、保護者入力ではなく医師診察で確認する項目として扱う。
- 牛乳アレルギーは患者画面で診断させず、ミルク変更、牛乳・乳製品、母の乳製品摂取と症状の関係を追加確認候補として扱う。

### 2-3歳: toddler

現MVPを維持する。

- 排便頻度
- 便の硬さ
- 排便時痛
- がまん
- 薬の飲み方
- 飲み忘れ、少なめ、中止、飲みにくさ
- トイレトレーニング
- 日誌サマリー

画面設計の方向:

- 現在の質問セットと条件分岐を壊さない。
- 年齢不明時は、当面このプロファイルを安全側の既定として扱う。

### 4歳以降: child

医療者レビュー用の確認候補:

- 園・学校で排便できるか
- 排便を我慢する場面
- 腹痛
- 便失禁、下着汚れ
- 夜尿症、昼間尿失禁の有無
- トイレ回避、羞恥心
- 生活リズム、朝の排便時間
- 偏食、野菜摂取、お菓子、食事、水分、運動
- 服薬自己管理、飲み忘れ
- 発達について相談中か
- 他院に通院中の病気、他院での診断、長期管理薬
- 便秘が怖くて薬をやめられない不安
- 頓用薬を家に置きたい希望は患者画面では聞かず、医師側確認事項として扱う
- 保護者と本人の認識差

画面設計の方向:

- 保護者回答だけでなく、本人が答えやすい表現も検討する。
- 学校生活での排便困難、便失禁、服薬継続の課題を拾う。
- 夜尿、昼間尿失禁、発達相談、他院通院中の病気、他院処方薬は、便秘との関係を決めつけず、医師が確認する背景情報として扱う。
- 便秘が怖くて薬をやめられないという希望は、実際の便性状や排便頻度と不安の内容を分けて整理する。
- 頓用薬を家に置きたい希望は、チェックを入れると薬がもらえるという誤解を避けるため、患者画面ではなく医師側確認事項として扱う。
- 医療側、患児、保護者の治療方針に一致性が見られない場合は、当院での治療継続困難または二次医療機関紹介の検討事項として扱う。
- 患者向けメモは、本人にも見せやすい言葉にする。

## ChatGPT相談用テキストの切り替え

全モード共通:

- 診断や処方の最終決定は医師が行う。
- 患者・保護者へ直接、薬の増減・中止・再開を指示しない。
- 不足情報と不確実性を明示する。

年齢層別に追加する観点:

| プロファイル | ChatGPTに追加する観点                                                                                                         |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| infant       | 出生時・1か月健診の指摘、哺乳、体重増加、嘔吐、腹部膨満、離乳食、牛乳・乳製品との関連、早期発症など、医師が確認すべき背景情報 |
| toddler      | 痛み、がまん、薬の飲み方、トイレトレーニング、日誌変化                                                                        |
| child        | 園・学校での排便、便失禁、夜尿、昼間尿失禁、腹痛、生活リズム、偏食、発達相談、他院通院中の病気、他院処方、服薬自己管理、継続不安、本人と保護者の認識差 |

治療方針検討モードでは、年齢層に応じた注意点を加えるが、具体的な処方量変更の数値指示は出させない。

詳細は `age_profile_chatgpt_prompt_design.md` にまとめる。

## Google Sheets設計

現時点で追加を検討する列:

| シート   | 列                | 用途                                         |
| -------- | ----------------- | -------------------------------------------- |
| patients | birth_date        | 既存。年齢自動計算に使う。                   |
| visits   | age_profile       | 受診時点での年齢プロファイル。将来追加候補。 |
| visits   | age_text_at_visit | 受診時点の表示年齢。将来追加候補。           |

すぐには列追加しない。まずは `patients.birth_date` から医師側履歴表示時に年齢を算出する現行仕様を維持する。

質問セット切り替えの詳細は `age_profile_questionnaire_switch_design.md` にまとめる。

## 実装順

1. 年齢プロファイル設計を確定する。
2. 2-3歳の既存質問セットを `toddler` として固定する。
3. 0-1歳向け問診プロトタイプを文書で作る。詳細は `infant_questionnaire_prototype.md`。
4. 4歳以降向け問診プロトタイプを文書で作る。詳細は `child_questionnaire_prototype.md`。
5. 医療者レビューを受ける。
6. レビュー後に、質問セット切り替えを実装する。詳細は `age_profile_questionnaire_switch_design.md`。
7. ChatGPT相談用テキストに `age_profile` 別の追加観点を入れる。詳細は `age_profile_chatgpt_prompt_design.md`。
