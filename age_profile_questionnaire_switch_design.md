# 年齢プロファイル別質問セット切り替え設計

## 目的

0-1歳、2-3歳、4歳以降で問診項目を切り替えられるようにする。

ただし、現MVPでは患者画面で年齢や生年月日を入力させない方針を維持する。患者画面の入力負担を増やさず、既存の2-3歳問診を壊さないことを優先する。

## 結論

初期実装では、患者画面に年齢選択や年齢入力を置かない。

年齢プロファイルは、以下の優先順で決める。

1. `patients.birth_date` と受診日から自動計算する。
2. 医師側で明示的に `age_profile` を上書きできるようにする。
3. 年齢が不明な場合は、現MVPの `toddler` 質問セットを既定として使い、医師側には `年齢未確認` と表示する。

この方式なら、初診で生年月日が未登録でも問診を止めずに済む。再診では、初診後に `patients.birth_date` を入力しておけば、次回以降は自動で年齢プロファイルを切り替えられる。

将来の実運用案として、初診時のみ受付または医師側で生年月日を登録する。患者本人・保護者がスマホ問診画面で生年月日を入力する運用にはしない。生年月日は直接識別性が高い情報として扱い、GitHub PagesのURL、QRコード、患者向けメモには含めない。

## 年齢プロファイル判定

| 年齢 | age_profile | 質問セット |
|---|---|---|
| 0歳0か月-1歳11か月 | `infant` | 0-1歳向け質問 |
| 2歳0か月-3歳11か月 | `toddler` | 現MVP質問 |
| 4歳0か月以上 | `child` | 4歳以降向け質問 |
| 年齢不明 | `unknown` | 現MVP質問 + 年齢未確認表示 |

`unknown` の場合、内部的な質問セットは `toddler` と同じにする。医師側サマリー、便秘履歴、ChatGPT貼り付け用テキストでは、年齢未確認として追加確認に出す。

## 画面別の設計

### 受付用QR作成ページ

現状:

- 患者IDと来院トークンから問診URLを作成する。

将来案:

- 原則として年齢は入力しない。
- 例外的に、初診で年齢層が明らかな場合だけ、詳細設定に `年齢層を指定` を置くことは検討可能。
- ただし、誤選択のリスクがあるため、初期実装では見送る。

推奨:

- 受付用QRは今のまま。
- 年齢層の判定はGoogle Sheetsの `patients.birth_date` 側に寄せる。
- 将来的に受付で入力する場合も、QR URLに生年月日を含めず、Google Sheetsの患者台帳へ登録する形にする。

### 初診時の生年月日登録

将来案:

- 初診時のみ、受付または医師側で `patients.birth_date` を登録する。
- 登録後は、再診時に患者IDと受診日から年齢プロファイルを自動判定する。
- 登録画面は患者スマホ側ではなく、院内端末または医師入力画面に置く。
- 生年月日はGitHub PagesのURL、QRコード、患者向けメモ、ChatGPT貼り付け用テキストには直接出さない。
- ChatGPT貼り付け用テキストには、生年月日ではなく `2歳4か月` のような年齢表示だけを使う。

運用上の利点:

- 患者入力の負担が増えない。
- 再診時に年齢が自動で更新される。
- 0-1歳、2-3歳、4歳以降の質問セット切り替えに使える。
- 生年月日をURLへ載せないため、公開URL運用と相性がよい。

### 患者問診ページ

現状:

- 年齢入力欄は削除済み。
- 患者ID、来院トークンがある場合にSheetsへ保存する。

将来案:

- 問診開始時にWeb Appへ患者IDを照会し、`age_profile` を取得できれば質問セットを切り替える。
- ただし、QR読み込みから初期表示までの待ち時間、通信失敗時の扱い、Web App公開範囲の問題がある。

推奨:

- 初期実装では、患者問診ページは今の `toddler` 質問セットを維持する。
- 年齢層別質問セットの実装は、医療者レビュー後に行う。
- 実装する場合も、通信失敗時は `toddler` 質問セットで続行できるようにする。

### 医師側履歴ページ

現状:

- `patients.birth_date` から直近受診日時点の年齢を表示する。

将来案:

- 年齢表示に加えて `age_profile` を表示する。
- 年齢未確認の場合は、上部に `年齢未確認: patients.birth_dateを入力してください` と出す。
- 必要時に医師側で `age_profile` を手動補正できる欄を設ける。

推奨:

- まず医師側履歴とChatGPT貼り付け用テキストに `age_profile` を表示する。
- 患者画面の質問切り替えより先に、医師側の確認表示を整える。

### 医師入力ページ

将来案:

- `patients.birth_date` 入力欄を設ける。
- `age_profile` 手動補正欄を設ける場合は、通常は折りたたみの詳細設定にする。

推奨:

- まずは `birth_date` 入力を医師入力または別の患者台帳編集画面に追加する。
- `age_profile` 手動補正は、運用上必要になった時点で追加する。

## データ設計

### 既存

`patients`

- `patient_id`
- `age_years`
- `age_months`
- `created_at`
- `note`
- `birth_date`

### 将来追加候補

`patients`

| 列 | 用途 |
|---|---|
| `age_profile_override` | 医師側で年齢層を手動補正する場合に使う。通常は空欄。 |
| `age_profile_note` | 補正理由や注意事項。 |

`visits`

| 列 | 用途 |
|---|---|
| `age_profile` | 受診時点で使った年齢プロファイル。 |
| `age_text_at_visit` | 受診時点の表示年齢。 |
| `questionnaire_version` | 質問セットのバージョン。 |

初期実装では、列追加は最小限にする。質問セット切り替えを実装する段階で `visits.age_profile` と `visits.questionnaire_version` を追加する。

## 質問セット管理

将来のJavaScript構成案:

```text
questionnaire.js
questionnaire-profiles.js
questionnaire-profile-infant.js
questionnaire-profile-toddler.js
questionnaire-profile-child.js
```

各プロファイルは、以下を持つ。

- 基本質問
- 条件分岐
- 日誌項目
- 医師向けサマリー項目
- 患者向けメモ項目
- ChatGPT追加観点

まずは現MVPの質問セットを `toddler` として固定する。0-1歳、4歳以降は文書レビュー後に追加する。

## 切り替えロジック案

```text
1. URLに patient_id がある
2. Web Appへ patient_id を照会する
3. birth_date があれば、受診日から年齢を計算する
4. age_profile を決める
5. 対応する質問セットを表示する
6. 通信失敗、birth_date未登録、age_profile不明の場合は toddler 質問セットで続行する
7. 保存時に age_profile と questionnaire_version を visits に残す
```

## 通信失敗時の方針

患者問診は止めない。

表示案:

```text
年齢情報を確認できませんでした。
このまま問診を続けられます。
診察時に年齢を確認します。
```

ただし、初期実装ではこの通信を行わないため、この表示は将来案とする。

## 実装順

1. 現MVPの質問セットを `toddler` として整理する。
2. 医師側履歴とChatGPT貼り付け用テキストに `age_profile` 表示を追加する。完了。
3. 医師側の患者台帳編集で `birth_date` を入力しやすくする。完了。
4. `visits.age_profile` と `visits.questionnaire_version` の保存を追加する。
5. `infant` と `child` の質問セットを実装する。
6. 患者問診ページで、Web App照会による質問セット切り替えを実装する。
7. 通信失敗時に `toddler` で続行するフォールバックを確認する。
