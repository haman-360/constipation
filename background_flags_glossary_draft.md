# background_flags 用語集ドラフト

## 目的

`background_flags` は、医師側の患者台帳で使う固定タグです。

自由記載の `background_history` とは分けて、以下に使います。

- 患者さんがQRコードを読み取った時に、変化しうる併存相談だけを確認表示する
- 毎回聞かなくてよい質問を省く
- 医師側履歴とChatGPT貼り付け用テキストで背景情報を安定して渡す
- 表記ゆれを減らす

## 基本方針

`background_flags` は、2種類に分けて扱います。

### 固定既往歴

後から基本的に変わらない情報です。

例: 出生歴、周産期歴、手術歴、小児外科紹介歴。

運用:

- 医師側履歴とChatGPT貼り付け用テキストには出す。
- 患者QR画面には原則出さない。
- 詳細は `background_history` に自由記載する。

### 変化しうる併存相談

治る、悪化する、終了する、フォロー先が変わる可能性がある情報です。

例: 夜尿、昼間尿失禁、発達相談、療育、他院通院、学校での配慮。

運用:

- 医師側履歴とChatGPT貼り付け用テキストに出す。
- 患者QR画面に表示し、変更や追加がある場合だけ入力してもらう。
- 一部タグでは、該当する毎回質問を省略候補にする。

## 運用ルール案

- `background_flags` には、この用語集にあるタグだけを使う。
- 詳細、病名、経過、担当科、薬剤名、出生週数などは `background_history` に自由記載する。
- 状態は `background_status` で、`継続中 / 過去にあり / 終了 / 不明` から選ぶ。
- 最後に確認した日を `background_updated_at` に入れる。
- 便秘との因果関係はタグだけでは断定しない。診察時に確認する背景情報として扱う。
- 患者QR画面では、固定既往歴や `background_history` の細かい内容は原則表示しない。

## background_history に書く情報

`background_history` は医師側の自由記載です。患者QR画面には原則表示しません。

例:

- 30週出生、出生体重980g、NICU入院歴あり。
- 重症新生児仮死、HIEで新生児期治療歴あり。
- 小児外科でヒルシュスプルング病は否定的と説明あり。
- 泌尿器科で夜尿症フォロー中。薬剤名は〇〇。

## 表記ゆれの扱い

以下のような表記は `background_history` には書いてよいですが、`background_flags` では固定タグへ寄せます。

| 書きがちな表現 | 寄せるタグ |
|---|---|
| 夜尿、夜尿症、夜のおねしょ | `夜尿あり` |
| 昼間尿失禁、日中尿もれ | `昼間尿失禁あり` |
| 昼間尿失禁でフォロー、排尿相談中 | `昼間尿失禁で相談中` |
| 発達相談、発達外来、発達相談中 | `発達について相談中` |
| 発達遅滞、発達特性、ASD、ADHD | `発達遅滞または発達特性あり` |
| 療育、児発、放デイ | `療育・発達支援を利用中` |
| 早産、低出生体重、極低出生体重、NICU | `早産・低出生体重・NICU入院歴` |
| 新生児仮死、重症新生児仮死、HIE | `新生児仮死・周産期合併症` |
| 生まれた時から便秘、生後すぐから便秘 | `出生直後から便秘` |
| ヒルシュスプルング病評価、低位鎖肛疑い、小児外科紹介 | `小児外科紹介・受診歴あり` |

## 患者QR画面の表示案

変化しうる併存相談だけを表示します。

例:

```text
登録済みの確認事項:
発達について相談中、昼間尿失禁で相談中
状態: 継続中
最終確認: 2026-05-09

変更や追加がある場合だけ入力してください。
```

固定既往歴や `background_history` の細かい内容は表示しません。

## レビューしたい点

- `夜尿あり` は患者QR画面に毎回出すか、4歳以降だけ出すか。
- `昼間尿失禁あり` と `昼間尿失禁で相談中` を両方残すか、片方に統合するか。
- `神経・筋疾患` と `便秘に影響しうる薬あり` は固定既往歴として患者QR非表示でよいか。
