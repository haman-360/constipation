## 重要な実装修正

### 1. 診察支援ダッシュボードを追加

従来は送信後画面の右側パネルに、テキスト形式の医師向けミニサマリーとJSONを表示していた。

今回、右側パネルを以下の順に読めるダッシュボードに変更した。

1. 緊急度/確認優先度
2. 排便スナップショット
3. 安全・追加確認
4. 便秘薬
5. 診察で見るポイント
6. AIが判断していないこと
7. 送信JSON

画面上の見出し:

```text
診察支援ダッシュボード
```

### 2. 医師レビュー用の構造化データ生成を追加

ロジック位置:

- `constipation-ai-mvp/src/questionnaire.js`
- `generatePhysicianReview(input)`

主な戻り値:

- `urgency`
  - `level`: `alert` / `watch` / `stable`
  - `label`
  - `message`
- `headline`
- `stool`
- `safety`
- `stoolConcerns`
- `medication`
- `checkItems`
- `notJudged`
- `raw`

既存の `generateSummary(input)` は残している。

コピー用には、画面表示のHTMLではなく従来のテキストミニサマリーを使うようにしている。

実装位置:

- `constipation-ai-mvp/src/app.js`
- `els.summaryOutput.dataset.copyText = generateSummary(payload);`

### 3. 送信後に「戻る / 送信する」が残るバグを修正

ユーザー提供スクリーンショットで、送信後にも左下のナビゲーションが表示されていた。

原因:

- HTMLの `hidden` 属性は付いていたが、CSSの `.nav { display: flex; }` が優先されていた可能性が高い。

修正:

- `constipation-ai-mvp/styles.css`

```css
[hidden] {
  display: none !important;
}
```

これにより送信後画面では不要なナビゲーションが消える想定。

## 現在のダッシュボード表示仕様

### 緊急度/確認優先度

`generatePhysicianReview()` 内の `reviewUrgency(data)` で判定。

#### alert

条件:

- 強い腹痛・お腹の張り
- 嘔吐あり
- 食欲が少ない
- 機嫌が悪い
- 食欲も機嫌も気になる

表示例:

```text
受付・スタッフ共有
安全確認あり + 4日以上無排便/間隔あり + 硬便 + 強い排便時痛 + がまん
強い腹痛・嘔吐・食欲/機嫌変化のいずれかがあります。
```

#### watch

条件:

- 4日以上無排便/4日以上あく
- 水様便
- 硬便・強い痛み・がまん
- 内服少なめ/忘れ/中止
- 内服困難

表示例:

```text
診察で確認
硬便 + がまん + 内服少なめ/忘れ/中止
回答に応じた追加確認項目があります。
```

#### stable

条件:

- 上記に該当しない

表示例:

```text
通常確認
目立つ追加確認なし
現時点で強い安全表示はありません。
```

## ユーザー確認済みスクリーンショット

ユーザーが以下2つの画面を確認。

### 通常確認例

入力の概要:

- 最終排便: 今日
- 頻度: わからない
- 硬さ: やわらかい
- 痛み: 少し痛がる
- がまん: ない
- 便秘薬: 便秘のお薬は使っていない

表示:

```text
通常確認
目立つ追加確認なし
現時点で強い安全表示はありません。
```

### 診察で確認例

入力の概要:

- 最終排便: 昨日
- 頻度: 2-3日に1回くらい
- 硬さ: 硬い
- 痛み: 少し痛がる
- がまん: ある
- お薬: ときどき忘れる
- 出血: 便に混じる
- 便付着: ときどきある
- 少なめ/忘れ/中止後: 出る間隔があいた、硬くなった

表示:

```text
診察で確認
硬便 + がまん + 内服少なめ/忘れ/中止
回答に応じた追加確認項目があります。
```

この方向性について、ユーザーから「いい感じですね」とコメントあり。

## 現在の役割境界

このプロトタイプは、診療判断AIではなく、診察前の問診整理AI。

すること:

- 保護者の回答を構造化する。
- 必要な追加質問だけを表示する。
- 医師が診察前に一瞬で読むための画面を作る。
- 安全面の確認がある場合、受付または医療スタッフ共有の表示を出す。
- 診察で確認するとよい観察情報を並べる。

しないこと:

- 診断。
- 便塞栓の有無判断。
- モビコールなどの処方量変更。
- 薬の増減、中止、再開の指示。
- 治療中止可否の判断。
- 専門紹介の要否判断。
