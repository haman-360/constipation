# 2-3歳便秘AI 医師レビュー用ワンページ

更新日: 2026-05-05

## 目的

2-3歳の慢性機能性便秘症の診察前に、保護者入力を短時間で構造化し、医師が状態変化を把握しやすい形で提示する。

本プロトタイプは診療判断AIではなく、待合問診を整理する「診察前情報整理・診察支援ツール」としてレビューをお願いしたい。

## 対象

- 2-3歳の慢性機能性便秘症
- モビコールなどのPEG製剤の導入期・維持期
- 医師から便性状を見ながら調節してよいと説明されているケースを含む

## 現在の運用イメージ

1. 受付で5桁の患者IDと来院トークン入りQRを作成する。
2. 保護者がiPhoneなどで問診を入力する。
3. 入力内容をGoogle Apps Script経由でGoogle Sheetsに保存する。
4. 医師は便秘履歴ページで、問診履歴、週次日誌、処方履歴、トイレトレーニング履歴を確認する。
5. 必要時、ChatGPT貼り付け用テキストで診察前の変化点や不足情報を整理する。
6. 処方履歴とトイレトレーニング履歴は、医師が必要時に別フォームから追記する。

## このAIがすること

- 保護者の回答を構造化する。
- 最終排便、排便頻度、便の硬さ、排便時痛、がまん、便秘薬の状況を整理する。
- 回答に応じて腹痛・張り、嘔吐、食欲・機嫌、出血、便付着、薬の飲みにくさ、少なめ・飲み忘れ・中止後の変化を追加確認する。
- 医師向けミニサマリー、診察前確認サマリー、患者向けメモを生成する。
- 強い腹痛、嘔吐、食欲と機嫌の変化は、受付共有ではなく医師が診察で確認する項目として整理する。
- 直近日誌がある場合、週次サマリーとして保存し、履歴ページで確認できるようにする。

## このAIがしないこと

- 診断
- 便塞栓の有無判断
- モビコールなどの処方量変更
- 薬の増減・中止・再開指示
- 治療終了判断
- 専門紹介判断
- 先天性疾患の有無判断

## 基本6問

| 項目 | 保護者向け質問 |
|---|---|
| 最終排便 | 最後にうんちが出たのはいつですか？ |
| 排便頻度 | 最近、うんちはどれくらいの間隔で出ていますか？ |
| 便の硬さ | 最近のうんちの硬さはどれに近いですか？ |
| 排便時痛 | うんちをするとき、痛がることはありますか？ |
| がまん | うんちをがまんしているように見えることはありますか？ |
| 便秘薬 | 便秘のお薬は、今どのように飲んでいますか？ |

## 条件付き追加確認

| 条件 | 追加で確認すること |
|---|---|
| 4日以上うんちが出ていない、または4日以上あくことがある | お腹の張り・腹痛、嘔吐、食欲・機嫌 |
| 水のような便（下痢に近い） | お腹の張り・腹痛、嘔吐、食欲・機嫌 |
| 硬い便、強い痛み、泣く・とても嫌がる、がまんあり | 血がつくこと、パンツやおむつに少し便がつくこと |
| お薬が飲みにくくて残る | 味、量、飲ませるタイミング、子どもの拒否など |
| お薬が少なめ、飲み忘れ、中止中 | その後の硬さ、間隔、痛みの変化 |

## 表示分類

| 表示 | 条件 | 意味 |
|---|---|---|
| 診察で確認 | 4日以上無排便/間隔、水様便、硬便・強い痛み・がまん、腹痛・張り、嘔吐、食欲・機嫌変化、内服少なめ/忘れ/中止、内服困難など | 診察時に追加確認したい情報あり |
| 通常確認 | 上記に該当しない | 現時点で目立つ追加確認項目なし |

保護者向けの緊急受診判断や受付共有表示は出さない。医師が診察で確認する運用とする。

## 履歴連携

Google Sheetsを、電子カルテ連携ではなく便秘診療の補助台帳として使う。

| シート | 入力元 | 役割 |
|---|---|---|
| `patients` | 問診送信時に自動登録 | 患者ID台帳 |
| `visits` | 保護者問診 | 受診・問診履歴 |
| `diary_weekly` | 保護者問診の日誌欄 | 直近日誌の週次サマリー |
| `prescriptions` | 医師入力 | 処方履歴 |
| `toilet_training` | 医師入力 | トイレトレーニング履歴 |

`prescriptions` と `toilet_training` は患者回答から推測せず、医師が必要時に追記する。

## 患者向け最終画面

- 院内保存中は、画面下部に「このままお待ちください」と表示する。
- 保存完了後は、「画面を閉じても大丈夫です」と表示する。
- 患者用メモは、必要な場合に自分用としてコピーできる。
- 患者用メモでは、薬の量を決める表現を避け、診察で相談する準備に限定する。

## 医師向け表示例

```text
診察で確認
食欲と機嫌の変化があります。4日以上の無排便または排便間隔のあき、硬い便、強い排便時痛、がまん行動も確認されています。
診察時に確認したい追加情報があります。

診察で見るポイント:
- 4日以上うんちが出ていない、または4日以上あくことがあるため、お腹の張り・腹痛、嘔吐、食欲・機嫌を診察で確認。
- 硬い便、強い痛み、がまんがあるため、出血や便が少しつく様子もあわせて確認。
- 便塞栓の有無や追加対応は、この画面では判断せず診察で確認します。

AIが判断していないこと:
診断 / 便塞栓の有無 / 処方量変更 / 治療中止可否 / 専門紹介の要否
```

## 医師レビューで確認したい点

- 基本6問で診察前の状態把握として不足がないか。
- 条件付き追加確認の条件と文言が実臨床に合うか。
- `診察で確認` / `通常確認` の表現と条件が適切か。
- 薬の自己調節、少なめ、飲み忘れ、飲みにくさの分け方が臨床実態に合うか。
- 患者用メモが薬の自己判断を促さないか。
- 処方履歴・トイレトレーニング履歴を医師入力台帳として扱う方針が妥当か。
- 便秘履歴ページに表示する履歴項目に過不足がないか。
- ChatGPT貼り付け用テキストに、診断・処方指示・紹介判断へ踏み込む表現がないか。

## 初回プロトタイプでは扱わない項目

- 周産期・新生児期の詳細
- 基礎疾患、発達・行動面、定期薬
- 家族歴
- トイレ詰まり
- 園・外出先での排便困難
- 浣腸歴、受診中断歴、便秘開始時期

これらは、医師レビュー後に必要性の高いものから追加する。
