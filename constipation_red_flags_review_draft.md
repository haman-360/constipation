# 小児便秘AI 診察で確認項目レビュー下書き

更新日: 2026-05-05

## 位置づけ

この文書は、2-3歳便秘AI MVPの医療者レビューで確認するためのたたき台です。

本MVPは診断、緊急受診判断、専門紹介判断を行わず、入力された情報を診察前に整理します。アプリ内では「レッドフラッグ」という語を保護者向けに表示せず、該当しうる情報は医師が診察で確認する項目として扱います。

## 現在の反映方針

- 「受付・スタッフ共有」は廃止する。
- 強い腹痛・お腹の張り、嘔吐、食欲と機嫌の変化は、医師が診察で確認する項目として扱う。
- 保護者向けの緊急受診判断表示は出さない。
- 発熱、胆汁性嘔吐、胎便排泄遅延、肛門周囲所見などは現MVPには追加しない。
- 強い腹部膨満は「お腹の張り」に含める。
- 便性状は「硬い / 普通（バナナくらい） / やわらかい / 水のよう（下痢に近い） / わからない」とする。

## 参照元

- NICE Quality standard QS62: Constipation in children and young people, Quality statement 1: Assessment
- NICE guideline CG99: Constipation in children and young people: diagnosis and management
- ESPGHAN/NASPGHAN guideline: Evaluation and Treatment of Functional Constipation in Infants and Children
- NIDDK: Symptoms & Causes of Constipation in Children

## 現MVPで診察確認に上げる項目

| 候補 | 現MVPの入力 | 現在の扱い | 監修確認 |
|---|---|---|---|
| 4日以上無排便または4日以上あく | `q1_last_bowel_movement = 4日以上前` / `q2_bowel_frequency = 4日以上あくことがある` | 診察で確認 | 追加質問の内容が妥当か |
| 水のような便 | `q3_stool_consistency = 水のよう（下痢に近い）` | 診察で確認 | 便秘治療中の軟便としての扱いと確認範囲 |
| 硬い便 | `q3_stool_consistency = 硬い` | 診察で確認 | 出血・痛み・がまんとの組み合わせ条件 |
| 強い排便時痛 | `q4_pain = 強く痛がる` / `泣く、またはとても嫌がる` | 診察で確認 | 保護者向け文言の強さ |
| がまん行動 | `q5_withholding = ある` / `強くある` | 診察で確認 | 年齢相応の行動との差 |
| 強い腹痛・お腹の張り | `q9_abdominal_symptom = 強い` | 診察で確認 | 腹痛と腹部膨満を分ける必要があるか |
| 嘔吐 | `q10_vomiting = ある` | 診察で確認 | 「吐いた」の範囲が広すぎないか |
| 食欲と機嫌の変化 | `q11_appetite_mood = 食欲も機嫌も気になる` | 診察で確認 | 食欲のみ・機嫌のみとの差を維持するか |
| 血がつく | `q7_blood = 紙に少しつく` / `便に混じる` | 診察で確認 | 表現や選択肢の分け方 |
| 便付着・便失禁 | `q8_soiling = ときどきある` / `よくある` | 診察で確認 | 便塞栓を示唆する表現に踏み込みすぎないか |
| 薬が少なめ、飲み忘れ、中止中 | `q6_med_status` | 診察で確認 | 薬の自己判断を促さない表現になっているか |
| 薬が飲みにくい | `q6_med_status = 飲みにくくて残る` | 診察で確認 | 味、量、タイミング、拒否の選択肢が実用的か |

## 現MVPでは入力しない候補

ESPGHAN/NASPGHANやNICEの評価項目に含まれるが、現MVPの待合問診では扱っていない項目です。今後の拡張または医師診察内確認の候補です。

| 候補 | 理由 | 現MVPでの扱い案 |
|---|---|---|
| 便秘開始が新生児期から | Hirschsprung病などを含む鑑別の手がかり | 医師診察で確認、必要なら初回登録に追加 |
| 胎便排泄遅延 | 新生児期情報で保護者記憶が曖昧な場合あり | 医師診察で確認 |
| 成長不良・体重減少 | 背景疾患・全身状態の手がかり | 医師診察で確認 |
| 発熱 | 便秘以外の体調変化 | 現MVPには入れない |
| 胆汁性嘔吐 | 受診優先度が高い可能性 | 現MVPには入れない |
| 肛門位置異常・瘻孔・肛門周囲所見 | 身体診察項目 | アプリでは扱わない |
| 神経学的異常・仙骨部所見 | 身体診察項目 | アプリでは扱わない |
| 家族歴 | 背景疾患の手がかり | 初回登録で扱うか検討 |

## 医療者レビューでの質問

- 現在の「診察で確認」条件は広すぎるか、狭すぎるか。
- 食欲低下のみ、機嫌不良のみを通常確認側に残す扱いは妥当か。
- 血がつく、便に混じるなどの出血表現をさらに分ける必要があるか。
- 発熱、体重減少、胆汁性嘔吐などを待合問診に追加すべきか。
- 「レッドフラッグ」という語をアプリ内に出さず、医師向けの診察確認項目として扱う方針でよいか。
- 医師レビュー前プロトタイプとして、現MVPは診療判断に踏み込んでいないか。
