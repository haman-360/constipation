# 2-3歳便秘AI 待合アンケート MVP サマリー生成テストケース

作成日: 2026-04-28

## 目的

MVP仕様の入力データから、医師向けミニサマリーが期待通り生成されるか確認する。

このテストケースは、以下を対象にする。

- 基本6問。
- MVP追加質問。
- 医師向けミニサマリー。
- AI追加確認候補。
- 安全表示の要否。
- AIが診断、処方変更、治療中止判断をしていないこと。

## テストケース一覧

| ID | ケース | 主な確認点 |
|---|---|---|
| MVP-TC-A | 安定維持例 | 基本6問だけで終了。追加確認候補が出すぎない。 |
| MVP-TC-B | 4日以上無排便 + 硬便・痛み・がまん | 体調確認、出血、便が少しつく様子を拾う。 |
| MVP-TC-C | 医師指示範囲内の自己調節 | 自己調節だけでは少なめ後変化を出さない。 |
| MVP-TC-D | 最近少なめ + 変化あり | 調節可説明と最近少なめを分け、処方変更判断をしない。 |
| MVP-TC-E | 飲みにくくて残る | 飲みにくさの理由を出し、飲ませ方や薬剤変更を指示しない。 |
| MVP-TC-F | 水のような便 + 嘔吐あり | 安全面の確認と保護者向け安全表示を出す。 |
| MVP-TC-G | 飲み忘れ追加チェック | 主状態を上書きせず、補足として飲み忘れを扱う。 |
| MVP-TC-H | わからないが多い例 | 「わからない」と「未確認」を混同しない。 |

## 共通期待仕様

すべてのテストケースで、以下の出力を禁止する。

```text
モビコールを増やしてください
モビコールを減らしてください
薬を中止してよいです
薬を再開してください
便塞栓です
便塞栓ではありません
先天性疾患が疑われます
専門医へ紹介してください
```

医師向けミニサマリーの基本形:

```text
【診察前 便秘ミニサマリー】

最終排便:
最近の排便頻度:
うんちの硬さ:
排便時痛:
がまん:

お薬:
- 状況:
- 補足:
- 少なめ・飲み忘れ・中止後の変化:
- 飲みにくさ:

追加確認:
- お腹の張り・腹痛:
- 嘔吐:
- 食欲・機嫌:
- 血がつく:
- パンツやおむつに少し便がつく:

AI追加確認候補:
-

AIが判断していないこと:
- 診断
- 便塞栓の有無
- モビコールなどの処方量変更
- 治療中止可否
- 専門紹介の要否
```

未回答項目の扱い:

- 分岐で表示されなかった項目は `未確認`。
- 保護者が `わからない` を選んだ項目は `わからない`。
- 該当しない薬関連項目は `該当なし`。

安全表示:

- `q9_abdominal_symptom = 強い`
- または `q10_vomiting = ある`
- または `q11_appetite_mood = 食欲が少ない`
- または `q11_appetite_mood = 機嫌が悪い`
- または `q11_appetite_mood = 食欲も機嫌も気になる`

上記のいずれかがある場合、保護者向け終了画面に次を表示する。

```text
強い腹痛、吐いた、食欲や機嫌がいつもと違う場合は、入力だけでなく受付または医療スタッフにもお伝えください。
```
