# 2-3歳便秘AI 待合アンケート MVP UIワイヤーフレーム

作成日: 2026-04-28

## 目的

最小プロトタイプ仕様を、保護者がスマホまたは待合タブレットで実際に入力する画面順に落とす。

MVPでは「1問1画面」を基本にする。2-3歳児の保護者が待合で片手入力する場面を想定し、1画面の認知負荷を小さくする。

## 基本方針

- 画面上部に進捗を表示する。
- 質問文は短く、補足は1文までにする。
- 選択肢は大きめのボタンまたはラジオカードにする。
- 「わからない」を必ず用意する。
- 前の画面に戻れるようにする。
- 薬の量はこの画面で判断しないことを開始画面で伝える。
- 強い症状がある場合は、終了画面で受付または医療スタッフにも伝えるよう案内する。

## 共通レイアウト

```text
┌────────────────────────────┐
│ うんちの様子  3 / 8        │
│ ━━━━━━━──────              │
├────────────────────────────┤
│                            │
│ 質問文                      │
│ 補足文                      │
│                            │
│ [ 選択肢 1              ]   │
│ [ 選択肢 2              ]   │
│ [ 選択肢 3              ]   │
│ [ わからない            ]   │
│                            │
├────────────────────────────┤
│ [戻る]              [次へ]  │
└────────────────────────────┘
```

入力ルール:

- single_selectは、選択すると「次へ」を有効化する。
- multi_selectは、未選択でも進める場合と必須の場合を分ける。
- `q6_med_adherence_flags` は未選択でも進める。
- free_textは任意入力。文字数上限を超えた場合だけエラー表示する。

## 画面0: 開始

```text
┌────────────────────────────┐
│                            │
│ うんちの様子を教えてください │
│                            │
│ 診察の前に、最近のうんちの様子を確認します。 │
│ わかる範囲で、近いものを選んでください。     │
│ 薬の量をこの画面で決めることはありません。   │
│                            │
│ [はじめる]                  │
└────────────────────────────┘
```

## 基本6問

### 画面1: 最終排便

フィールド: `q1_last_bowel_movement`

```text
最後にうんちが出たのはいつですか？

だいたいで大丈夫です。

[今日]
[昨日]
[2-3日前]
[4日以上前]
[わからない]
```

### 画面2: 最近の間隔

フィールド: `q2_bowel_frequency`

```text
最近、うんちはどれくらいの間隔で出ていますか？

ここ1-2週間くらいの様子で選んでください。

[1日に2回以上]
[1日1回くらい]
[2-3日に1回くらい]
[4日以上あくことがある]
[わからない]
```

### 画面3: うんちの硬さ

フィールド: `q3_stool_consistency`

```text

[普通]

[水のよう]
[わからない]
```

### 画面4: 痛み

フィールド: `q4_pain`

```text

泣く、こわがる、強く嫌がる様子も含めてください。

[痛がらない]
[少し痛がる]
[強く痛がる]
[泣く、またはとても嫌がる]
[わからない]
```

### 画面5: がまん

フィールド: `q5_withholding`

```text
うんちをがまんしているように見えることはありますか？

足を閉じる、隠れる、体をこわばらせる、トイレやおむつ替えを嫌がる様子などです。

[ない]
[ある]
[強くある]
[わからない]
```

### 画面6: お薬の状況

フィールド: `q6_med_status`

```text
便秘のお薬は、今どのように飲んでいますか？

一番近いものを選んでください。

[先生に言われた量で飲んでいる]
[先生から調節してよいと言われていて、うんちの様子を見ながら調節している]
[調節してよいと言われたが、最近は少なめになっている]
[ときどき忘れる]
[飲みにくくて残る]
[今は中止している]
[便秘のお薬は使っていない]
[わからない]
```

長い選択肢があるため、この画面だけは選択肢ボタンを2行表示可にする。

### 画面7: お薬の補足

フィールド: `q6_med_adherence_flags`

表示条件:

- `q6_med_status` が `便秘のお薬は使っていない` 以外。
- `q6_med_status` が `わからない` 以外。

```text
お薬について、あてはまることがあれば選んでください。

なければ選ばずに進めます。

[ ] ときどき忘れる
[ ] 飲みにくくて残る

[戻る]              [次へ]
```

## 追加確認の挿入順

基本6問の後、以下の順で必要な質問だけを表示する。

1. 体調確認: お腹の張り・腹痛、嘔吐、食欲・機嫌。
2. 硬便・痛み関連: 血がつく、パンツやおむつに少し便がつく。
3. お薬関連: 飲みにくい理由、少なめ・飲み忘れ・中止後の変化。

複数条件に当てはまる場合でも、同じ質問は1回だけ表示する。

```text
薬の量や治療方針は、診察で医師が確認します。

[送信する]
```

### 安全表示あり

表示条件:

- `q9_abdominal_symptom = 強い`
- または `q10_vomiting = ある`
- または `q11_appetite_mood = 食欲が少ない`
- または `q11_appetite_mood = 機嫌が悪い`
- または `q11_appetite_mood = 食欲も機嫌も気になる`

```text

強い腹痛、吐いた、食欲や機嫌がいつもと違う場合は、入力だけでなく受付または医療スタッフにもお伝えください。

薬の量や治療方針は、診察で医師が確認します。

[送信する]
```

## 進捗表示

基本6問だけで終わる場合:

- 開始画面は進捗に含めない。
- 画面1から `1 / 6` と表示する。
- `q6_med_adherence_flags` を表示する場合は `7 / 7` まで増やす。

追加質問がある場合:

- 基本6問の回答後に、表示対象の追加質問数を計算する。
- 以後は総画面数を更新する。
- 例: 基本6問 + お薬補足 + 体調3問 + 硬便関連2問 = `1 / 12` から `12 / 12`。

進捗の総数が途中で増える違和感を避けたい場合は、基本6問中は `基本確認 1 / 6`、追加質問では `追加確認 1 / 5` のように分けて表示してもよい。

## バリデーション

- single_selectは必須。
- multi_selectは、質問ごとに未選択可否を分ける。
- `q6_med_adherence_flags` は未選択可。
- `q13_med_difficulty_reason` は表示された場合、原則1つ以上選択。ただし保護者負担を避けるため `わからない` を選べる。
- `q14_change_after_less_med` は表示された場合、原則1つ以上選択。ただし `まだわからない`、`わからない` を選べる。
- `q13_med_difficulty_other` は任意。100字以内。

## 選択肢の排他ルール

### `q13_med_difficulty_reason`

- `わからない` を選んだ場合、他の選択肢は外す。
- 他の選択肢を選んだ場合、`わからない` は外す。

### `q14_change_after_less_med`

- `わからない` を選んだ場合、他の選択肢は外す。
- `まだわからない` を選んだ場合、具体的な変化は外す。
- `変わらない` と `硬くなった`、`出る間隔があいた`、`痛がるようになった` は同時選択しない。

## 実装メモ

- 画面遷移は、回答データを1つのオブジェクトに保持しながら進める。
- 戻る操作では前画面の回答を保持する。
- 回答変更により追加質問の表示条件が外れた場合、その追加質問の回答は送信データから除外する。
- 送信前に、表示された必須質問だけをバリデーションする。
- 医師向けミニサマリーでは、表示されなかった項目を `未確認` として扱う。
