# 医療者監修依頼チェックリスト

## 目的

2-3歳の慢性機能性便秘症診療サポートツールについて、診療判断AIではなく「待合問診・診察前情報整理ツール」として安全に使えるかを医療者に確認してもらう。

## 監修者に最初に共有する前提

- このツールは診断しない。
- 便塞栓の有無を判断しない。
- 処方量変更、薬の増減・中止・再開、治療終了、専門紹介を判断しない。
- 保護者入力を構造化し、医師が診察前に確認しやすい形へ整理する。
- 保護者向けには不安をあおらず、必要時に受付・スタッフへ伝える案内に留める。

## 実装反映メモ

以下の監修コメントを実装へ反映済み。

- 受付・スタッフ共有は削除。
- 保護者向け安全表示は削除。
- 強い腹痛・お腹の張り、嘔吐、食欲と機嫌の変化は、医師が診察で確認する項目に統一。
- 発熱、胆汁性嘔吐、胎便排泄遅延、肛門周囲所見などは現MVPには追加しない。
- 便性状は `硬い / 普通（バナナくらい） / やわらかい / 水のよう（下痢に近い） / わからない` に更新。
- 医師向けの「便塞栓」表現は維持。

## 監修で決めたいこと

### 1. 受診・共有基準

現在の分類が妥当か確認する。

> いずれも、**ロジック変更が必要** です。削除で良い。受付やスタッフは、これらの内容を確認しません。すべて、私（医師）が診察で確認するため。

| 入力条件                         | 現在の扱い         | 監修で決めたいこと                                                                     |
| -------------------------------- | ------------------ | -------------------------------------------------------------------------------------- |
| 強い腹痛・お腹の張り             | 受付・スタッフ共有 | この条件でよいか。腹痛と腹部膨満を分けるべきか。                                       |
| 嘔吐あり                         | 受付・スタッフ共有 | 「吐いたことがありますか？」で広すぎないか。反復嘔吐、胆汁性嘔吐などを別に聞くべきか。 |
| 食欲も機嫌も気になる             | 受付・スタッフ共有 | 食欲低下のみ、機嫌不良のみを「診察で確認」に留めてよいか。                             |
| 4日以上無排便、または4日以上あく | 診察で確認         | 受付共有へ上げる組み合わせ条件が必要か。                                               |
| 水のような便                     | 診察で確認         | 便秘治療中の軟便として扱う範囲と、受付共有へ上げる条件をどう分けるか。                 |
| 硬便、強い痛み、がまん           | 診察で確認         | 出血・便付着の追加確認条件として妥当か。                                               |
| 血がつく                         | 診察で確認         | 「紙に少し」 と「便に混じる」で扱いを分けるべきか。                                    |
| 便付着・便失禁                   | 診察で確認         | 便塞栓を示唆する表現に踏み込みすぎていないか。                                         |
| 薬が少なめ、飲み忘れ、中止中     | 診察で確認         | 薬の自己判断を促さない表現になっているか。                                             |
| 薬が飲みにくい                   | 診察で確認         | 質問項目と選択肢が実臨床に合うか。                                                     |

### 2. 現MVPに追加すべきか決めたい項目

> 発熱、胆汁性嘔吐、胎便排泄遅延、肛門周囲所見など、は、**ロジック変更が必要（削除）**。体重増加不良・体重減少、は、**そのままでよい**。強い腹部膨満は、お腹の張りに含める（**ロジック変更が必要**）。

| 候補                                   | 追加するか   | コメント                                       |
| -------------------------------------- | ------------ | ---------------------------------------------- |
| 発熱                                   | 要確認       | 受付共有候補として聞くか。                     |
| 胆汁性嘔吐、緑色の嘔吐                 | 要確認       | 保護者向けにどう表現するか。                   |
| 便秘開始が新生児期から                 | 要確認       | 初回登録で聞くか、診察内確認に留めるか。       |
| 胎便排泄遅延                           | 要確認       | 待合問診で聞くには難しすぎないか。             |
| 体重増加不良・体重減少                 | 要確認       | 保護者入力より医師確認に回すか。               |
| 強い腹部膨満                           | 要確認       | 現在の「強い腹痛・お腹の張り」に含めてよいか。 |
| 肛門周囲所見、神経学的所見、仙骨部所見 | 原則アプリ外 | 身体診察項目として扱う方針でよいか。           |

### 3. 保護者向け文言の確認

> 前述の通り、**ロジック変更が必要（削除）**

現在の安全表示:

```text
強い腹痛、吐いた、食欲も機嫌もいつもと違う場合は、受付または医療スタッフにもお伝えください。
```

確認したいこと:

- 「強い腹痛」は保護者に伝わるか。
- 「吐いた」は広すぎるか。繰り返す、緑色、ぐったりなどを入れるべきか。
- 「食欲も機嫌もいつもと違う」は適切か。
- 「受付または医療スタッフにもお伝えください」は強すぎないか、弱すぎないか。
- 「すぐ受診」「救急」などの言葉をこのツールで使わない方針でよいか。

### 4. 便性状分類の最終仕様

> **ロジック変更が必要**。選択肢は、これでOKだが、バナナくらい、の表現は入れてOK。「普通（バナナくらい）」の表現でよいのでは？画像・イラストは不要。「水のよう（下痢に近い）」といった表現は、あり。

現在の選択肢:

```text
硬い / 普通 / やわらかい / 水のよう / わからない
```

確認したいこと:

- 2-3歳の保護者向けにこの4分類で十分か。
- Bristol Stool Form Scale 風の7分類に寄せるべきか。
- 「ころころ」「バナナくらい」「どろどろ」「水のよう」など、保護者向け表現に変えるべきか。
- 画像・イラストを使うべきか。
- 「水のよう」を診察で確認に上げる現在の扱いが妥当か。

### 5. 医師向け表示の表現確認

> **そのままでよい**。見るのは私だけなので、このままでOK。

確認したい文言:

```text
便塞栓の有無や追加対応は、この画面では判断せず診察で確認します。
```

確認したいこと:

- 医師向けとして「便塞栓」という語を使ってよいか。
- 保護者画面には出さない方針でよいか。
- 「追加対応」という語が曖昧なら、別表現に変えるべきか。

## 監修者にお願いしたい返答形式

以下の3段階で返してもらうと実装に反映しやすい。

1. そのままでよい
2. 文言修正が必要
3. ロジック変更が必要

ロジック変更が必要な場合は、次の形で返してもらう。

```text
条件: 例）嘔吐あり + 強い腹痛
現在: 受付・スタッフ共有
修正案: 受付・スタッフ共有のままでよい / 診察で確認へ下げる / 追加質問を出す
保護者向け文言案: ...
医師向け文言案: ...
```
