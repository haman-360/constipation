# 医療者監修依頼チェックリスト

更新日: 2026-05-05

## 目的

2-3歳の慢性機能性便秘症診療サポートツールについて、診療判断AIではなく「待合問診・診察前情報整理ツール」として安全に使えるかを医療者に確認してもらう。

## 監修者に最初に共有する前提

- このツールは診断しない。
- 便塞栓の有無を判断しない。
- 処方量変更、薬の増減・中止・再開、治療終了、専門紹介を判断しない。
- 保護者入力を構造化し、医師が診察前に確認しやすい形へ整理する。
- 強い腹痛・お腹の張り、嘔吐、食欲と機嫌の変化などは、受付共有ではなく医師が診察で確認する項目として整理する。
- 保護者向け出力は、薬の自己判断を促さず、診察で相談する準備に限定する。

## 現在の実装概要

- GitHub Pages固定URLから、受付用QR作成ページ、問診ページ、医師用履歴URL作成ページを開く。
- 受付用QR作成ページで、5桁の患者IDと来院トークン入りの問診QRを作成する。
- iPhoneでQRを読み取り、保護者が問診を入力する。
- 送信後、Google Apps Script経由でGoogle Sheetsに保存する。
- 患者向け最終画面では、画面下部に院内保存中・保存完了・保存失敗を表示する。
- Google Sheetsでは、`patients` / `visits` / `diary_weekly` を患者入力から保存する。
- `prescriptions` / `toilet_training` は患者回答から推測せず、医師が必要時に手入力する台帳として扱う。
- 便秘履歴ページでは、問診履歴、処方履歴、トイレトレーニング履歴、週次日誌、ChatGPT貼り付け用テキストを確認できる。
- ChatGPT貼り付け用テキストは、診察前の変化点、不足情報、確認観点の整理に使う前提で、診断・処方判断はさせない。

## 監修で確認したいこと

### 1. 基本問診の妥当性

| 項目 | 現在の質問 | 確認したいこと |
|---|---|---|
| 最終排便 | 最後にうんちが出たのはいつですか？ | 選択肢の粒度が診察前確認として十分か |
| 排便頻度 | 最近、うんちはどれくらいの間隔で出ていますか？ | 4日以上あく選択肢の扱いが妥当か |
| 便の硬さ | 最近のうんちの硬さはどれに近いですか？ | `普通（バナナくらい）` などの表現が保護者に伝わるか |
| 排便時痛 | うんちをするとき、痛がることはありますか？ | 強い痛み、泣く・嫌がる表現が適切か |
| がまん | うんちをがまんしているように見えることはありますか？ | 2-3歳の行動として聞き方が妥当か |
| 便秘薬 | 便秘のお薬は、今どのように飲んでいますか？ | 少なめ、飲み忘れ、中止中、飲みにくさの分け方が実臨床に合うか |

### 2. 条件付き追加確認の妥当性

| 条件 | 追加で確認すること | 確認したいこと |
|---|---|---|
| 4日以上うんちが出ていない、または4日以上あく | お腹の張り・腹痛、嘔吐、食欲・機嫌 | 待合問診で聞く内容として過不足がないか |
| 水のような便（下痢に近い） | お腹の張り・腹痛、嘔吐、食欲・機嫌 | 便秘治療中の軟便としての扱いと、診察確認の範囲が妥当か |
| 硬い便、強い痛み、がまん | 血がつく、パンツやおむつに少し便がつく | 出血・便付着の聞き方が適切か |
| 薬が飲みにくくて残る | 味、量、タイミング、拒否など | 内服困難の選択肢が実用的か |
| 薬が少なめ、飲み忘れ、中止中 | その後の硬さ、間隔、痛みの変化 | 自己調節を促さない表現になっているか |

### 3. 表示分類の確認

現在の表示は `診察で確認` / `通常確認` の2分類とする。`受付・スタッフ共有` は廃止済み。

| 表示 | 意味 | 確認したいこと |
|---|---|---|
| 診察で確認 | 診察時に追加確認したい情報がある | 条件が広すぎる、または狭すぎるものがないか |
| 通常確認 | 目立つ追加確認項目はない | `問題なし` と誤解されない表現になっているか |

### 4. 医師向け表示の確認

確認したい文言:

```text
便塞栓の有無や追加対応は、この画面では判断せず診察で確認します。
```

確認したいこと:

- 医師向けとして「便塞栓」という語を使ってよいか。
- 保護者画面には出さない方針でよいか。
- 「追加対応」という語が曖昧なら、別表現に変えるべきか。
- 診断、処方量変更、治療中止、専門紹介へ踏み込む表現が残っていないか。

### 5. 患者向け最終画面の確認

現在の方針:

- 送信後、院内保存中は画面下部に「このままお待ちください」と表示する。
- 保存完了後は「画面を閉じても大丈夫です」と表示する。
- 患者用メモは、必要な場合のみ自分用にコピーできる。
- 患者用メモでは薬の量を決めず、診察で相談する内容を整理する。

確認したいこと:

- 保護者が途中で閉じにくい案内になっているか。
- 保存失敗時の案内が不安をあおりすぎないか。
- 患者用メモが薬の自己判断を促さないか。

### 6. 医師入力台帳の確認

`prescriptions` と `toilet_training` は医師側入力のみとする。

| 台帳 | 入力者 | 用途 |
|---|---|---|
| 処方履歴 | 医師 | 処方日、薬剤名、量、指示内容、医師メモの記録 |
| トイレトレーニング履歴 | 医師 | 記録日、開始状況、おむつ・パンツ、トイレ拒否、メモの記録 |

確認したいこと:

- 患者回答から自動推測しない方針でよいか。
- 診察前履歴として表示する項目に過不足がないか。
- 日時つきで保存する運用が妥当か。

## 監修者にお願いしたい返答形式

以下の3段階で返してもらうと実装に反映しやすい。

1. そのままでよい
2. 文言修正が必要
3. ロジック変更が必要

ロジック変更が必要な場合は、次の形で返してもらう。

```text
対象: 例）嘔吐あり
現在: 診察で確認
修正案: このままでよい / 追加質問を出す / 入力項目から外す
保護者向け文言案: ...
医師向け文言案: ...
```
